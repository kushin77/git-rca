# Backlog (Top-priority Stories)

This backlog provides an initial breakdown of high-priority features and user stories derived from the epics. Each item should be created as a GitHub issue and estimated during grooming.

1) Story: Create repository skeleton and CI
- Epic: Core Platform & Architecture
- Description: Add README, CONTRIBUTING, LICENSE, basic CI pipeline that builds a placeholder service.
- Acceptance Criteria: CI runs and displays passing build for skeleton.

2) Story: Implement Git Event Connector (DEV)
- Epic: Data & Integrations
- Description: Ingest git push/pull events into the dev data store sample schema.
- Acceptance Criteria: Sample events are stored and queryable.

3) Story: Design Investigation Canvas mockups
- Epic: RCA Workflow & UX
- Description: Low-fidelity designs for investigation canvas and timeline.
- Acceptance Criteria: Two pilot engineers validate mockups.

4) Story: Baseline Threat Model
- Epic: Security, Compliance & Privacy
- Description: Create threat model and initial mitigations list for MVP.
- Acceptance Criteria: Threat model documented and triaged.

5) Story: Pilot onboarding guide
- Epic: Adoption, Docs & Enablement
- Description: Step-by-step guide to onboard a pilot team and run first RCA.
- Acceptance Criteria: Guide tested with one pilot user.

Prioritization: Start with 1->2->3 then 4->5 for MVP.
